# Правила работы с проектом

Руководство для студентов по работе с учебным проектом **FACEIT CS2 Dashboard**.

---

## Формат коммитов

Каждый коммит должен начинаться с префикса и номера темы:

```
feat(theme-1): добавить радарную диаграмму
fix(theme-4): исправить утечку памяти в usePlayerData
style(theme-2): добавить shimmer-анимацию
test(theme-6): написать тесты для getStatColor
```

### Допустимые префиксы

| Префикс    | Назначение                              |
|------------|-----------------------------------------|
| `feat`     | Новая функциональность                  |
| `fix`      | Исправление бага                        |
| `style`    | Визуальные изменения (CSS, анимации)    |
| `test`     | Добавление или исправление тестов       |
| `refactor` | Рефакторинг без изменения поведения     |
| `docs`     | Изменения в документации                |

---

## Создание ветки

Формат имени ветки: `theme-N/short-description`

```bash
git checkout -b theme-1/radar-chart
git checkout -b theme-4/player-search
```

---

## Проверка работы

Перед каждым коммитом обязательно выполните:

1. **`npm run typecheck`** — проверка типов (ошибок быть не должно)
2. **`npm test`** — тесты (если работаете с темами 6-7)
3. **Визуальная проверка в браузере** — откройте `localhost:5173`
4. **Проверить dark mode** — переключите тему
5. **Проверить мобильную версию** — сузьте окно браузера

---

## Правила

1. **Dark mode** — все новые элементы должны поддерживать тёмную тему (используйте `dark:` классы Tailwind)
2. **Mobile responsive** — интерфейс должен корректно отображаться на мобильных устройствах
3. **Не трогайте готовые компоненты** — файлы в `components/core/`, `components/charts/`, `hooks/`, `utils/` уже готовы и протестированы
4. **Работайте только в `features/`** — ваш код пишется в файлах внутри `src/features/theme-*/`
5. **Ищите TODO маркеры** — каждое задание отмечено `// TODO: Задание X.Y` с подсказками

---

## Запуск одного теста

```bash
npm test -- colors.test       # только тесты цветов
npm test -- useTheme.test     # только тесты темы
npm test -- integration       # только интеграционные тесты
```

## Tailwind CSS v4

Проект использует **Tailwind CSS v4** — конфигурация через `@import "tailwindcss"` в CSS. Файл `tailwind.config.js` не нужен.

## Полезные расширения VS Code

- **Todo Tree** (Gruntfuggly) — показывает все TODO и BONUS маркеры
- **Tailwind CSS IntelliSense** — автодополнение классов
- **ESLint** — подсветка ошибок линтера
- **Prettier** — форматирование кода
