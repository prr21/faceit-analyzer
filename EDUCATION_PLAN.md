# Карта прогресса — FACEIT CS2 Dashboard

## Обзор курса

Практический курс по веб-разработке на базе реального аналитического дашборда FACEIT CS2. Студенты дописывают логику в подготовленных местах, отмеченных `// TODO:` маркерами.

- **7 тем**, **10 заданий**, **46 TODO маркеров** + **3 BONUS**
- Стек: React 19, TypeScript, Vite, Tailwind CSS 4, ECharts 5, Express.js 5, Vitest
- Два трека: фронтенд (без сервера) и полный стек

## Карта зависимостей

```
Трек A — Фронтенд (работает сразу, без сервера)
  Тема 1 (компоненты) ─┐
  Тема 2 (анимации)  ──┤── независимые, в любом порядке
                        │
Трек B — Полный стек    │
  Тема 3 (сервер)  ─────┤
       │                │
       ├── Тема 4 (fetch, поиск)
       │        │
       │        └── Тема 5.2 (автообновление)
       │
       └── Тема 5.1 (сравнение)

Тестирование (параллельно с любым треком)
  Тема 6 (unit-тесты)
  Тема 7 (интеграционные тесты)
```

- **Трек A** не требует сервера — компоненты работают с предзагруженными данными
- **Трек B** начинается с Темы 3 (сервер), затем Темы 4–5 используют его API
- **Тестирование** можно выполнять параллельно с любой темой

## Критерии оценки

### Тема 1 — Frontend-компоненты
- [ ] 1.1: Радарная диаграмма рендерится с данными по картам (WR, KD, ADR)
- [ ] 1.2: Клик по строке матча раскрывает/скрывает детальную карточку
- [ ] 1.3: 4 карточки (лучшая/худшая WR, лучшая KD, лучшая ADR) с фильтрацией < 3 матчей

### Тема 2 — Мультимедиа и анимации
- [ ] 2.1: Скелетон-карточки пульсируют (shimmer), счётчик плавно нарастает от 0
- [ ] 2.2: Изображения карт загружаются, при ошибке — fallback с буквой. SVG-иконки рядом со статистикой

### Тема 3 — REST API сервер
- [ ] Сервер стартует на порту 3000
- [ ] `GET /api/player/:nickname` возвращает данные игрока
- [ ] CORS настроен (запросы с localhost:5173 проходят)
- [ ] Rate limiting работает (лишние запросы отклоняются)

### Тема 4 — Асинхронное API
- [ ] 4.1: Данные загружаются динамически по URL `?player=nickname`, спиннер при загрузке
- [ ] 4.2: Поиск с debounce (300мс), запрос отправляется при 3+ символах, результаты отображаются

### Тема 5 — Динамическое обновление
- [ ] 5.1: Два игрока сравниваются side-by-side (ELO, WR, KD, лучшая карта), один может не загрузиться
- [ ] 5.2: Прогресс-бар в шапке, автообновление по интервалу, кнопка ручного обновления

### Тема 6 — Unit-тестирование
- [ ] Все тесты проходят (`npm test`)
- [ ] Тесты `getStatColor` покрывают граничные значения для WR/KD/ADR/HS%
- [ ] Тесты `useTheme` проверяют переключение и сохранение в localStorage

### Тема 7 — Интеграционное тестирование
- [ ] Все интеграционные тесты проходят
- [ ] Корректный моккинг данных и рендеринг компонентов
- [ ] Тесты проверяют навигацию по табам, отображение данных, взаимодействие

## Рекомендации для преподавателя

1. **Todo Tree** — установите расширение в VS Code, оно покажет дерево всех TODO и BONUS маркеров с навигацией
2. **Два трека** — студенты могут начать с Трека A (фронтенд) без настройки сервера
3. **API-ключ** — для Тем 3–5 студентам нужен FACEIT API ключ ([developers.faceit.com](https://developers.faceit.com/))
4. **BONUS** — дополнительные задания для продвинутых студентов (3 штуки), оцениваются отдельно
5. **Dark mode + Mobile** — все новые элементы должны поддерживать тёмную тему и мобильную вёрстку
6. **Подробные описания** — каждая тема имеет отдельный файл в `docs/themes/theme-N-*.md` с теорией, документацией и пошаговыми инструкциями
7. **Dev-режим** — при `npm run dev` приложение показывает mock-данные, сервер не нужен для Трека A
